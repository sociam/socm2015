Abstract

This file is an example of the use of the css classes for the paper submission to the SOCM2015 workshop. It is meant to be straight-forward, please select "view source" for examples. It defines classes for a depth of 2 levels (section and subsection), figures, and tables. Please make use of links as much as possible, including internal references. Bibliographic references should be in ACM bib style - see an example in the References section.

Introduction

Sarven, who's work I make a mishmash of here, has raised a call for linked research which we attempt to respond to with the submission and publication process at the 3rd edition of the SOCM workshop.

This template can be used with Sarven's acm.css and lncs.css styles, to generate printable views similar to the respective formatting styles.

Use this page as a template for your submission. Please make use of links as much as possible, including internal references.

Sections

Sections are defined with a <h2> tag containing the title of the section, and a <div>. The order in which the 2 tags appear is only important for the print views, as follows: the sections for which the <div> tag is first will not be numbered if the style requires numbering, e.g. the abstract, the keywords, the references, the acknowledgements; the sections for which the <h2> tag is first will be numbered if the style requires numbering

An optional id attribute can be used in the <h2> tag, as in internal reference to the section. The abstract, keywords, references, are all represented as sections. They should be included for the acm and lncs styles to work.

Subsections

Subsections are made just as sections, but with decreasing h tags: <h3>. <h4>, etc. Subsections can have id attributes as well.

Numbering

Sections and subsections should not be manually numbered. If they need to be referenced from text, an internal link should be set and used, making the numbering unneccessary. The print styles add automatic numbering to sections and subsections, so adding section numbers can lead to duble numbering when these styles are used. See the section on Sections on how the numebring can be triggered.

Figures, tables, etc.

Figures and tables should be given ids so that they are linkable. Figures and tables should have descriptive captions.

To create a captioned figure, use a <div class="figure"> tag containing the <img>, and the caption in a separate <p class="caption"> or <div class="caption"> tag. Images can be as big as needed, but they will be resized to a maximum width of 800px.

Bibliography links

The references should be added in a consistent style. For the SOCM workshop please use ACM citation style. Please use links in the text where possible instead of references for any online resources. Published papers and articles should however be added in the references section and referenced like this [1].
